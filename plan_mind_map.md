# 漫画翻译项目技术方案

## 技术选型总览

### 管线7阶段
- 1-文本检测：YOLOv8-OBB（主力）+ DBNet（辅助）
- 2-图像增强：TACE三级级联增强
- 3-OCR识别：PP-OCRv5 + SVTR
- 4-机器翻译：CADGT五阶段翻译管线（Qwen2.5-VL-2B底层驱动）+ 两阶段混合调度
- 5-图像修复：LaMa（Large Mask Inpainting）
- 6-译文渲染：SAAT风格感知自适应排版渲染
- 7-GUI交互：QT6 PySide6

## 六大创新点

### 创新一：文本检测 — 旋转框-分割双模检测
- 简写：Oriented & Segmentation Dual-Path Detection
- 核心思路：YOLOv8-OBB主力负责竖排+倾斜文本；DBNet辅助负责弧形+变形文本；双路结果融合
- YOLOv8-OBB擅长：竖直文本、倾斜气泡文本，用旋转矩形框+多角度数据增强
- DBNet擅长：弧形文字、严重变形文字（拟声词ドドド等），用文本区域分割+可微分二值化
- 创新价值：现有工具均单一检测器，对竖向文本和弧形文本兼顾不足；双管线互补填补空白

### 创新二：图像增强 — TACE三级自适应级联增强
- 简写：Three-Tier Adaptive Cascade Enhancement
- 级别1：CLAHE对比度 + Unsharp Mask锐化（传统CV，<1ms），置信度够→直接交付OCR
- 级别2：SVTR原生抗模糊识别，训练时已做模糊增强
- 级别3：置信度不足→Real-ESRGAN文本框定向超分→重试OCR
- 关键设计：不对全图超分，只对文本框小区域按需增强
- 正常图像零额外开销，模糊图像精准投入

### 创新三：OCR识别 — SVTR多语种漫画适配
- 简写：SVTR Comic-Adaptive Multilingual Recognition
- 架构升级：CRNN(CNN+RNN串行) → SVTR(全Transformer并行)
- 推理速度提升30%+，对变形/艺术字体鲁棒性更强
- 内置方向分类器实现竖排文本自动校正
- 针对中日英漫画文本做专属微调

### 创新四：机器翻译 — CADGT角色感知对话图翻译（核心创新）
- 简写：Character-Aware Dialogue Graph Translation
- 核心理念：将漫画翻译建模为"微型话剧翻译"，不再逐气泡独立翻译
- 阶段1-页面理解：气泡文字定位 + 角色归属(尾巴检测) + 场景分类(战斗/日常/恋爱/严肃) + 视觉特征提取
- 阶段2-对话图构建：日漫阅读顺序解析(右→左、上→下) + 气泡按角色分组串联 + 对话边关系建立
- 阶段3-角色画像：自称词提取(オレ→热血/僕→温和/わたし→文静) + 句尾习惯分析 + 角色关系推断 → 输出人设卡
- 阶段4-上下文感知翻译：五维翻译上下文 = 场景维度+角色维度+历史维度+对话维度+视觉维度
- 阶段4-两阶段混合调度：简单气泡→OPUS-MT快速直译(~50ms，覆盖60%)；复杂气泡→Qwen2.5-VL-2B多模态翻译(~3s，覆盖40%)
- 阶段5-一致性校验：同角色语气一致性+对话衔接+术语统一→不一致则回溯重译
- 创新价值：翻译单位从单气泡→整页对话图；角色感知从无→自动提取人设+语气适配

### 创新五：图像修复 — LaMa FFT全局感受野快速修复
- 简写：LaMa-based Global Context Inpainting
- 原理：基于傅里叶变换(FFT)在频域建模，全局感受野处理大面积文字去除
- 优势：对漫画网点(halftone周期性纹理)还原效果显著；大蒙版友好；模型200MB，CPU约200ms/区域
- 对比：传统OpenCV仅靠像素扩散，无法处理网点纹理/速度线/渐变背景

### 创新六：译文渲染 — SAAT风格感知自适应排版渲染
- 简写：Style-Aware Adaptive Typesetting
- 两种输出形态：气泡内替换渲染PNG（原漫画阅读体验）+ 纯文本对照表（校对/学习/字幕）
- 阶段1-风格检测：原文字颜色(K-Means聚类)、描边(Canny边缘)、阴影(亮度梯度)、字号(框高÷行数)、字重(二值化占比)、对齐方式
- 阶段2-空间计算：气泡轮廓提取约束文字不溢出 + 尾巴位置避开 + LaMa修复区域确认
- 阶段3-自适应排版（核心算法）：确定排版方向(竖排→转横排)；二分搜索最佳字号[8pt,48pt]；智能断行(中日文按字+标点规则，英文按词)；行距字距微调(单行0/双行×0.3/多行×0.2) + 垂直居中
- 阶段4-分层风格渲染：第1层阴影→第2层描边→第3层主体文字→第4层叠加原图
- 渲染策略：译文短→字号放大、译文长→字号缩小+多行、空间紧→最小10pt兜底
- 全部传统CV实现，零模型依赖

## 各方案选择理由

### 文本检测：为何YOLOv8-OBB+DBNet
- 日漫30-40%文本为竖向，水平框会框入大量非文字区域 → OBB旋转框天然适配
- 拟声词(ドドド)弯曲变形，旋转框也无法精确包裹 → DBNet分割补充
- OBB共享YOLOv8全部预训练权重和工具链；DBNet是PaddleOCR内置模块
- 不选RT-DETR：训练资源需求大，不适合大学生；不选CRAFT：推理慢，不适合实时交互

### 图像增强：为何TACE
- 漫画70%+文字区域清晰（出版级），全图超分纯属浪费
- CLAHE是OpenCV内置一行代码，对轻度模糊立竿见影
- SVTR的Transformer自注意力比CNN+RNN对噪声更鲁棒
- Real-ESRGAN仅极度模糊兜底，只对几十像素宽文本框做超分

### OCR识别：为何SVTR替代CRNN
- CRNN是2015方案(CNN+RNN串行)，SVTR是2022方案(纯Transformer并行)
- 漫画文字变形是对"整体结构"破坏，CRNN逐片看易错，SVTR全局看可辨识
- SVTR-Tiny仅5MB，CPU毫秒级推理；PP-OCRv5已将SVTR作为官方推荐
- 不选EasyOCR：日文精度仅~80%，竖排几乎不支持
- 不选TrOCR：模型过大300MB+，CPU太慢

### 机器翻译：为何CADGT替代NMT
- 彻底改变翻译范型：现有工具全部"检测→识别→逐句翻译→渲染"，CADGT先理解整页对话结构
- 对话图建模角色间对话关系，不像传统逐句割裂
- 角色画像可持续累积：同一部漫画读得越多翻译越准
- 五维上下文交叉验证，歧义场景做正确风格选择
- 不选纯NMT：完全无上下文理解，同一角色三句话三种语气
- 不选纯SLM：缺乏结构化理解，不知气泡在页面上位置和角色

### 图像修复：为何LaMa
- FFT傅里叶变换是漫画修复天然优势：网点(halftone)是周期性点阵，频域建模天然强
- 蒙版通常比实际文字大（膨胀确保覆盖），LaMa设计初衷就是大面积缺失处理
- 不选OpenCV：网点纹理填充模糊一团，速度线明显断裂

### 译文渲染：为何SAAT
- 渲染是用户体验最终出口，前面再好渲染差=用户第一感知差
- 漫画文字视觉风格是信息：加粗=吼叫、小字=耳语、红字=愤怒、描边=强调
- 二分搜索自适应字号是关键技术：8-48pt范围自动找最大可放入字号
- 气泡轮廓约束比矩形约束更精准：漫画气泡不规则(椭圆/爆炸/云朵)
- 全部传统CV，零模型依赖

### GUI：为何PySide6
- 整个管线Python生态，PySide6零跨语言开销
- Python调试效率远高于C++编译链接
- 性能瓶颈在算法不在界面渲染
- pyinstaller一键打包单exe

## 技术路线图

### 输入→输出全流程
- 输入：漫画图像
- 步骤1-文本检测与页面理解：YOLOv8-OBB + DBNet → 文本框坐标 + 场景分类 + 角色归属
- 步骤2-TACE增强：三级自适应增强（按需触发）→ 增强后文本区域
- 步骤3-OCR识别：PP-OCRv5 + SVTR（漫画微调）→ 文本字符串 + 置信度
- 步骤4-CADGT翻译：对话图构建 → 角色画像 → 五维上下文翻译 → 两阶段混合调度 → 一致性校验 → 最终译文
- 步骤5-LaMa修复：原文字区域去除 + 背景重建 → 干净图像
- 步骤6-SAAT渲染：风格检测 → 空间计算 → 自适应排版 → 分层渲染 → 双输出
- 输出：翻译后漫画PNG + 文本对照表

## 模块划分

### 第一层-数据层
- Module1-模型管理器：模型懒加载 + 版本管理 + ONNX导出缓存 + 角色画像JSON持久化

### 第二层-算法管线层
- Module2-检测模块Detector：YOLOv8-OBB推理 + DBNet推理 + 双路NMS融合 + 气泡尾巴角色归属 → 输出TextBox数组
- Module3-增强模块Enhancer(TACE)：一级CLAHE传统CV + 二级SVTR透传 + 三级Real-ESRGAN定向超分 → 输出增强文本区域
- Module4-识别模块Recognizer：文本方向检测校正 + SVTR中日英多语种推理 + 置信度反馈触发三级回退 → 输出文本+语种+置信度
- Module5-翻译模块Translator(CADGT核心)：页面理解 + 对话图构建 + 角色画像 + 五维上下文翻译引擎 + 两阶段混合调度 + 一致性校验 → 输出结构化译文
- Module6-修复模块Inpainter(LaMa)：蒙版膨胀生成 + FFT全局感受野背景重建 → 输出去文字后干净图像
- Module7-渲染模块Renderer(SAAT)：风格检测->空间计算->自适应排版->风格渲染->双输出

#### Module7子模块详解
- 7.1-风格检测StyleDetector：K-Means颜色聚类 + Canny描边检测 + 亮度梯度阴影 + 字号字重对齐估计（全部纯CV）
- 7.2-空间计算SpaceCalculator：气泡轮廓约束 + 尾巴避开 + LaMa修复区域确认
- 7.3-自适应排版TypeSetter(核心)：确定排版方向 → 二分搜索最佳字号[8pt-48pt] → 智能断行(中文按字+标点规则/英文按词/混合) → 行距字距微调+垂直居中
- 7.4-风格渲染Painter：阴影(底层)→描边(中层)→主体文字(上层)→叠加原图(顶层)，按层次顺序绘制
- 7.5-双输出Exporter：渲染图PNG + 角色对照文本表，共享同一份CADGT译文

### 第三层-界面层
- Module8-GUI模块：主窗口(图像预览+控制面板) + 语言设置/框选翻译/一键翻译/文字擦除/手动排版/字体调节/预览编辑 + GUI线程与推理线程分离 + 进度条

### 第四层-工具支撑层
- Module9-工具模块：数据增强(模糊噪声旋转透视) + 标注辅助工具 + 微调脚本 + 评估脚本(mAP/OCR准确率/BLEU) + 配置管理

### 模块依赖关系
- Module1模型管理器→提供所有模型实例
- Module2检测 + Module4识别 + Module5翻译 三路并行
- 检测→Module3增强→Module4识别→Module5翻译（前半段串联，后半段汇聚）
- Module5翻译→Module6修复 + Module7渲染 并行→Module8 GUI
- Module9横跨支撑所有模块

## 可行性评估

- M2检测-YOLOv8-OBB+DBNet：成熟，Ultralytics/PaddleOCR开源 → 🟢低风险
- M3增强-CLAHE+Real-ESRGAN：成熟，OpenCV/Real-ESRGAN开源 → 🟢低风险
- M4 OCR-PP-OCRv5+SVTR：成熟，PaddleOCR官方 → 🟢低风险
- M5翻译-CADGT+Qwen2.5-VL-2B：组合创新，各组件独立开源 → 🟡中风险
- M6修复-LaMa：成熟，HuggingFace/IOPaint → 🟢低风险
- M7渲染-自适应排版+风格检测：纯算法，PIL/Cairo/OpenCV → 🟢低风险
- M8 GUI-PySide6：成熟，QT官方 → 🟢低风险
- M5风险可控原因：对话图和角色画像支持降级运行，最坏退化为逐气泡翻译

### 资源可行性
- 训练算力：需GPU微调，可用Colab免费T4(16G显存)或学校服务器 → 需确认

- 预训练模型：全部开源，HuggingFace/PaddleOCR下载 → 可行
- 磁盘空间：全部模型约3-4GB → 可行

### 团队可行性
- 5人够用，模块拆分后每人职责明确无中间地带

### MVP裁剪策略
- P0必做：检测+OCR+基础翻译+修复+基本GUI → 不可裁剪

### 最大风险与应对
